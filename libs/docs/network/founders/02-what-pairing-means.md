# Transport vs data access

**Transport pairing** (invite code, Noise, Hyperswarm) only proves you are talking to a specific device identity. It adds the peer to **My Network** so encrypted streams are allowed.

**Data access** is separate: another device does not automatically read your sparks. You explicitly **Add admin** in **Spark settings** so biscuits + DEK keyshares authorize decrypt + merge for that spark only.
